2.1python中一切皆对象

什么是对象？

官方文档说：在python中，对象是指在内存中具有唯一的一个标识符，类型和值的实列。

听不懂啊！说人话解释下:他是一个具有属性和方法的实体，这些方法和属性可以被访问和操作

好像有点听懂了但是没全懂， 那么我们通过案列释下。

all_object.py

所以说num是一个对象，其实类，函数都是对象

打印了这个函数 print_info的内存地址，因为是对象所以有个特点就是可以赋值给一个变量的

所谓的对象就是保存在内存中的一段数据包括(函数，类，变量，数据结构)

上述的对象都可以将其添加到数据结构中这里使用是列表，也可以用集合将list()换为set()即可

通过装饰器的方式讲解函数作为一个参数进行传递以及返回一个函数对象

这里解释下装饰器在python中属于一个语法糖，作用是在不更改一个函数的内部代码的前提下还可以给当前的函数添加一个额外的功能

其实装饰器简单点去使用在print_wrapper() 函数上添加[修饰符(Decorators)]()@my_wrapper`即可

可以看到两次的运行结果是一致的

综上所述对象有4个特点：

​ 1.赋值给一个变量

​ 2.可以添加到集合对象

​ 3.可以作为参数传递给函数

​ 4.可以当函数的返回值

2.2type,object,class之间的关系

type_object_class.py

探究对象是被谁创建的

发现 int这个类型本质是一个类

上一节说python中一切皆对象，对象是由类创建出来的，那么类又是谁创建的呢？

由此我们知道 int这个类是由``type`创建的，所以我们可以总结：

python中的数据类型是由类型类创建出来的

python中的类型是由type创建出来的

所以python中所有的对象是由type创建的

探究python中的object

object 是 所有类的基类

在python3使用__bases__查询自定义的类如果没有写继承关系则默认继承object，有继承关系的则按照继承关系的显示

这里就比较意思了发生了一个闭环：

​ object 是基类没有父类，type创建了object

​ type的父类是object，type创建了type

所以 type创建了所有对象包括(类对象)以及自己 ，object是所有类的基类包括(type)。

​ 1.list

​ 列表是可变序列，通常用于存放同类项目的集合（其中精确的相似程度将根据应用而变化）。

​ 从左至右分别是0，1，2… 从右至左分别是-1，-2，-3…

​ 使用切片访问列表元素 listname[start: end: step]

​ 增加

​ append(),extend() 二者的同异性：append()方法会将字符串、列表或元组当做一个整体作为要添加元素列表的一个元素，形成一个包 含字符串、列表或元组的“新列表”（“新列表”指的是列表内存地址没变，元素更新了）；而extend()方法会将它们包含的元素逐个添加 到列表中，不会将它们视为一个整体。

​ 插入

​ insert() 与append(),extend()不同的是可以指定开始插入的索引位置，+ 二者的区别是：使用+加法运算符将两个或者多个列表相加， 达到拼接列表的目的，这样做会生成一个新的列表，原有的列表不会被改变。

​ 从执行效率上看，不建议使用此种方法，因为执行效率较低

​ 删除

​ del关键字,pop()方法，remove(),clear()

​ 修改

​ listname[index] = element

​ listname：要修改元素的列表。

​ index：要修改元素对应的索引值。

​ element：新元素值。

​ 查询

​ listname.index(obj, start, end)

​ listname：表示列表名称。

​ obj：表示要查找的元素。

​ start：表示起始位置。可以不写，表示检索从开头到end（不包括end）的元素。

​ end：表示结束位置。可以不写，表示检索从start到末尾的元素。

​ start和end可以都不写，表示检索整个列表；都写，表示检索start和end（不包括end）之间的元素。

​ listname.count(obj)

​ count()方法的返回值：count()方法返回某个元素在列表中出现的次数。如果返回 0，表示列表中不存在该元素。

​ listname：表示列表名。

​ obj：表示要统计的元素。
